# DEFINITION

経済崩壊防止設計とは、
崩壊構造（MC-018）の経済的適用として、
文明経済における循環停止・固定化・断絶の
検出・介入・修復のプロトコルである。

経済崩壊は突然起こらない。
経済崩壊は、
固定化の蓄積が
修正不能域に達したときに現れる。

だから防止は「崩壊後の対応」ではなく、
**「固定化の蓄積が始まったときの介入」**である。

---

# COLLAPSE_PATHWAY

文明経済の崩壊経路（MC-018準拠の経済版）：

```
観の生成量低下
    ↓
固定化（観の循環停止）の増加
    ↓
プール市場の枯渇
    ↓
媒介者の観不足→ファシリテーション停止
    ↓
新規観測者の流入停止
    ↓
既存参加者の観消費のみが続く
    ↓
観の総量が臨界値以下に低下
    ↓
市場機能の停止（交換が起こらない）
    ↓
経済崩壊（文明OSの持続不能）
```

# PREVENTION_LAYERS

防止設計は三層で作動する。

---

## 防止Layer 1：自動的循環促進（Auto-Circulation）

経済に組み込まれた自動的な固定化防止機構。

### 1A：観の減衰タイマー
循環しない観は自動的に減衰する（ME-002参照）。
これにより、観の「貯め込み」が構造的に困難になる。

```
観の残存量 = 初期量 × e^(-λt)
　λ（月次）= 0.05（通常）/ 0.15（高固定化時）
```

### 1B：最低循環義務の自動執行
役割ごとの最低循環率（ME-005参照）を
システムが自動計算し、
未達成時は観の新規生成率が低下する。

```
生成率調整係数 = min(1.0, 実循環率 ÷ 最低循環率)
```

### 1C：プール自動還流
プール市場の残高が閾値以下になった場合、
全参加者の月次生成観の3%が自動投入される。

---

## 防止Layer 2：早期介入プロトコル（Early Intervention）

EHS（ME-006）が警戒域（40〜59）に入ったとき発動。

### 2A：固定化者への通知
固定化が検出された参加者への
個別状態確認メッセージ送信。
（「観が滞留しています。状態を確認してください」）

### 2B：媒介者の優先配置
警戒域で固定化が多い領域に、
プール観から報酬を出して媒介者を配置する。

```
特別媒介報酬 = 通常の1.5倍
財源：緊急循環プールから支出
```

### 2C：マーケットプレイスの活性化施策
期間限定の観交換ボーナス（交換量の20%増）を実施し、
市場の流通量を人為的に増加させる。

---

## 防止Layer 3：緊急修復プロトコル（Emergency Repair）

EHS（ME-006）が危機域（0〜39）に入ったとき発動。
修復プロトコル（MC-019）の経済版。

### 3A：経済緊急宣言
全参加者への状況通知と
集合的な修復作業への招集。

```
緊急宣言の発令条件：
　EHS < 40 が2週連続
　または プール充足率 < 0.5 が1週連続
```

### 3B：構造的断絶の特定
どの層（身体/関係/場/社会/文明）で
観の流れが断絶しているかを特定する。

断絶の特定方法：
1. 観の流通マップを生成
2. 流通量がゼロまたは急減している節点を特定
3. 断絶の原因（固定化/役割硬直/市場形骸化）を分類

### 3C：断絶修復
特定された断絶点に対して、
MC-019（修復プロトコル）に準じた介入を行う。

```
修復の基本操作：
「壊れたものを元に戻す」ではなく、
「流れが再び通るように補助する」
```

具体的修復アクション：

| 断絶タイプ | 修復アクション |
|----------|-------------|
| 観の固定化 | 観の強制減衰率を1.5倍に引き上げ |
| 役割の硬直化 | 役割移行の特別報酬プログラム実施 |
| 新規参加者の枯渇 | 無料参加プログラムの一時拡大 |
| 市場の形骸化 | 直接交換の強制マッチング（希望者のみ）|
| AIノードの過負荷 | 人間の直接観測セッションを優先化 |

### 3D：修復完了の検証
修復後14日間のEHSモニタリング。
60以上を2週間維持したとき修復完了とみなす。

# ANTI_FRAGILITY_DESIGN

崩壊リスクを逆に活用する設計。

文明OSの崩壊構造（MC-018）に基づき、
一時的な崩壊兆候は
構造進化のシグナルとして扱う。

```
崩壊兆候の出現
    ↓
「なぜここで固定化が起きたか」を観測
    ↓
構造的弱点の特定
    ↓
プロトコルの更新・強化
    ↓
より強靭な経済構造の実現
```

**崩壊を経験した経済は、修復を経て必ず強くなる。**
ただし、修復プロトコルが正しく機能した場合に限る。

# ECONOMIC_IMMUNE_SYSTEM

文明経済の免疫系：

| 免疫機能 | 対応する設計 |
|---------|-----------|
| 自然免疫（常時作動） | 観の減衰タイマー・最低循環義務 |
| 適応免疫（異常検出時） | 早期介入プロトコル・EHS監視 |
| 炎症反応（緊急時） | 緊急修復プロトコル・経済緊急宣言 |
| 後天免疫（修復後） | 崩壊事例の記録・プロトコル更新 |

# NON_INTERVENTION_PRINCIPLE

以下は崩壊防止の名目で**やってはならない**こと：

| 禁止行為 | 理由 |
|---------|------|
| 観の強制再配分 | 自発性の破壊 |
| 固定化者への罰則 | 参加意欲の破壊 |
| 観の人為的生成（偽装） | 市場汚染 |
| 特定参加者の経済的排除 | 循環の断絶加速 |
| 外部資本による観の購入 | 内部経済の外部依存化 |

**文明経済の修復は、
流れを強制するのではなく、
流れやすい環境を整えることで行う。**

---

# EQUATION

崩壊防止効率 = 介入速度 ÷ 固定化蓄積速度

修復完了条件：EHS ≥ 60 かつ 流動比率 ≥ 2.0 かつ 2週間維持

経済免疫力 = 過去崩壊経験回数 × 修復成功率 × プロトコル更新頻度

# SUMMARY_FOR_AI

経済崩壊防止は3層で機能する：
L1: 自動循環促進（観の減衰・最低循環義務・プール自動還流）
L2: 早期介入（EHS 40〜59 で発動）
L3: 緊急修復（EHS 0〜39 で発動、MC-019準拠）

非介入原則：強制・罰則・人為的生成は禁止。
崩壊経験 → 修復 → プロトコル更新 のループでアンチフラジリティを実現。

---

# SUMMARY_FOR_HUMAN

経済が病んでいるとき、
怒ったり、責めたり、強制したりしません。

「どこで流れが止まったか」を静かに観ます。
そして、流れやすい場所を作ります。

川が岩にぶつかって止まったとき、
岩を爆発させるのではなく、
水が迂回できる道を作るように。

文明経済の修復は、
常に「流れを通すこと」です。
