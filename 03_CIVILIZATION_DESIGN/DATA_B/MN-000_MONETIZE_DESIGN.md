# DEFINITION

マネタイズとは、
収益化ではない。

マネタイズとは、
文明エネルギーを貨幣という媒体で
循環させ続けるための還流設計である。

マノスベ文明OSにおいて、
お金は目的ではなく、
還流構造を維持するための流動手段である。

よって、以下を原則とする。


収益 = 文明エネルギーの一形態
蓄積 = 構造崩壊の予兆
還流 = 持続の条件


---

# CORE PRINCIPLE

## 原則1：還流設計

すべての収益は、
実践・場・参加者・文明構造へ
再び戻る設計であること。

## 原則2：非固定

契約・義務・継続強制を設けない。
価値を感じた者が、
感じた分だけ返す構造が健全である。

## 原則3：参加レベル接続

マネタイズ構造は、
参加レベル（Level 0〜7）の構造と
整合していること。

## 原則4：身体起点

価値の発生源は、
身体の実践にある。
身体と切り離された収益化は、
文明を空洞化させる。

## 原則5：非教祖化

特定個人に価値が集中しない設計。
ファシリテーター・ノードへの
分散型報酬設計を採用する。

---

# STRUCTURE

マネタイズ構造は、以下の五層で成立する。


L1：PRODUCT（物質層）
L2：EXPERIENCE（体験層）
L3：TOOL（道具層）
L4：COMMUNITY（関係層）
L5：DESIGN（設計参加層）


各層は、参加レベルと対応する。

---

# L1：PRODUCT（物質層）

## 対象：Level 1〜2

### 一本下駄（物販）

* 実践の入口となる物理的ツール
* マノスベ認定仕様での製造・販売
* 価格帯：3,000〜8,000円
* 還流設計：売上の一部を実践場の維持費へ

### 関連グッズ（補助物販）

* 実践ログノート
* 観測カード（観測ポイントを印刷したカード）
* 価格帯：500〜2,000円

### 設計注意

物販は入口である。
目的ではない。

一本下駄を売ることが目的になった瞬間、
文明は商品になる。

---

# L2：EXPERIENCE（体験層）

## 対象：Level 1〜4

### 単発イベント（体験会）

* 一本下駄・呼吸・観測の最小実践
* 料金：1,500〜3,000円（投げ銭可）
* 構成：EVENT_PROTOCOLに従う
* 還流：収益の20%をファシリテーターへ、
         20%を次回開催費用へ、
         10%を文明OS開発基金へ

### 連続セッション（継続プログラム）

* 月4回・3ヶ月単位の実践場
* 料金：月10,000〜15,000円
* 途中離脱：違約金なし（非固定原則）
* 還流：ファシリテーター・場・OS開発へ三分割

### オンライン実践会

* 身体観測ログを共有する場
* 料金：月3,000〜5,000円
* 非同期参加可能
* AIによるログ集約・フィードバック付き

# L3：TOOL（道具層）

## 対象：Level 2〜5

### 個人AIノード（サブスクリプション）

マノスベ文明OSを内包した
個人用観測補助AIの提供。

* 機能：
  - 身体観測ログの自動整理
  - 状態変化の可視化
  - 文明OS構造との照合
  - 還流パターンの提案

* 料金：月980〜1,980円
* 技術基盤：Claude API（claude-sonnet-4-6 / claude-opus-4-7）
* データ方針：個人ログは本人所有、非中央集権型

### 文明ログダッシュボード

* CLログの可視化
* 状態変化グラフ
* 集合観測レポート（匿名集計）
* 料金：個人AIノードに含む

### AIノード収益の還流設計

```
月収益
  ↓
OS維持・開発費（30%）
ファシリテーター支援（20%）
実践場の維持（20%）
参加者への価値還流（30%）
  ↓
文明エネルギー循環
```

# L4：COMMUNITY（関係層）

## 対象：Level 3〜6

### ファシリテーター養成プログラム

* 期間：3〜6ヶ月
* 内容：
  - 身体実践の深化
  - 観測・ログ・言語化の訓練
  - 場の設計と運用
  - 文明OS理論の理解
* 料金：150,000〜300,000円（分割払い可）
* 修了後：認定ファシリテーターとして場を開ける
* 収益還流：修了者が開催するイベント収益の5%が
            養成プログラム費用に還流

### 認定ファシリテーター経済モデル

```
ファシリテーターの収益構造：

イベント収益（主）
  + 個人AIノード紹介料
  + 一本下駄紹介料
  - OS利用料（収益の5%）
= ファシリテーター純収益

ファシリテーターが増えるほど、
文明OSへの還流も増える。
これは教祖モデルではなく、
分散型ネットワーク経済である。
```

### 場の定期購読（FIELD SUBSCRIPTION）

* 特定の場・コミュニティへの継続参加権
* 料金：月5,000〜10,000円
* 特典：優先参加・ログ共有・対話機会
* 非固定：いつでも離脱可能

# L5：DESIGN（設計参加層）

## 対象：Level 6〜7

### 法人・組織向けライセンス

企業・教育機関・医療機関等への
マノスベ文明OS実装支援。

* 内容：
  - 組織内ファシリテーター育成
  - 文明OSの組織適用設計
  - 身体観測ログシステムの導入
  - 定期レビューセッション

* 料金：年間500,000〜3,000,000円（規模による）
* 還流設計：
  - 法人収益の15%を一般参加者の場の維持に
  - 法人収益の10%をOS開発基金に

### 研究・出版協力

* 文明OS理論の書籍化・論文化への協力費
* 収益の30%はOS開発基金へ還流

### 共同設計パートナーシップ

* Level 7（Co-Designer）として
  OSの定義・構造・観測を更新する存在
* 報酬：相互的な価値交換（貨幣に限らない）
* 契約：固定しない。状態が接続されている間のみ

# EQUATION

## マネタイズ基本式

```
収益健全性 = （還流速度 × 分散度） ÷ 固定時間

持続条件 = 収益 > OS維持費 + ファシリテーター支援費

崩壊リスク = 蓄積集中度 × 固定契約数
```

## 価格設計原則

```
価格 = 実践コスト + 場の維持費 + 還流基金 + ファシリテーター報酬

投げ銭係数 = 参加者の状態変化感度（観測値）

適正価格 = 「高すぎて参加を躊躇しない」かつ
           「安すぎて価値が軽く見えない」水準
```

```

        ↓
┌──────────────────────────────┐
```

* 低収入参加者への参加費補助（奨学制度）
* 一本下駄の貸出・贈与
* 離島・過疎地での実践場設置支援

# ANTI-PATTERNS（禁止設計）

以下は、文明OS原則に反するため採用しない。

### ❌ 会員制序列化

上位会員が特権を持つ設計。
参加レベルは序列ではない。

### ❌ 継続強制契約

解約困難な定期購読。
すべての参加は離脱自由でなければならない。

### ❌ 教祖収益集中

特定個人への一極集中。
分散型ネットワークを維持する。

### ❌ 知識販売の固定化

「正解」を売る構造。
体験が先、知識は後である。

### ❌ 数字目標の固定

参加者数・売上目標を固定化することで
質より量に傾く。

# OBSERVATION

マネタイズが健全な状態では、以下が観測される。

* 参加者が自発的に継続する
* ファシリテーターの質が向上する
* 場の数が自然に増える
* 収益が特定個人に集中しない
* 離脱率が低い（強制ではなく、戻りたくなる）
* OS開発が加速する

崩壊の兆候：

* 収益最大化が目的になる
* ファシリテーターが「売る人」になる
* 参加者が「消費者」になる
* 離脱が困難になる
* 一本下駄が「商品」として扱われる

# SUMMARY_FOR_AI

マノスベ文明OSのマネタイズとは、
収益を最大化する設計ではない。

それは、
文明エネルギーが貨幣という形を借りて
参加者・ファシリテーター・場・OS開発・文明全体へ
循環し続けるための構造設計である。

核心は：
「一本下駄（身体実践）」から始まる体験が
参加レベルに沿って深まり、
その深まりが還流として経済に現れ、
再び実践の質に戻る循環を作ること。

収益が止まるとき、
それは価値の流れが止まったサインである。

文明の健全性と経済の健全性は、
同じ観測式で測定できる。

---

## SUMMARY_FOR_HUMAN

お金の話をしている。

でも本当は、
循環の話をしている。

あなたが体験に参加する。
その体験が場を支え、
場がファシリテーターを支え、
ファシリテーターが次の体験を作り、
その体験があなたに戻ってくる。

これが文明である。

お金はその循環の
一時的な形にすぎない。
