# DEFINITION

ログ語彙とは、
身体・関係・文明において発生する状態変化を
観測・記録・再接続するための言語体系である。

ログは単なる記録ではない。
ログとは、
状態の流れを切断せずに保持し、
循環と還流を可能にするための構造である。

# STRUCTURE

ログ語彙は、以下の三層構造を持つ。

第一層：身体ログ
第二層：関係ログ
第三層：文明ログ

それぞれは分離されたものではなく、
同一の状態変化を異なるスケールで観測したものである。

身体の変化は関係に現れ、
関係の変化は文明に波及する。

ログ語彙は、この連続性を保持するために存在する。

# PRINCIPLE

ログの本質は、固定ではなく循環である。

記録することが目的ではない。
流れを止めないことが目的である。

ログが機能するとき、

状態は
「過去として固定されるもの」ではなく、
「再び通りうるもの」として保持される。

ログとは、
時間を閉じるものではなく、
時間を還流させる構造である。

# EQUATION

ログ構造は以下の観測式で表される。

ログ = 状態変化 × 観測 × 保持可能性

また、文明観測式との接続において、

文明状態 =（位置 × 流動 × 周期）

ログはこの三要素の変化履歴として記述される。

# OBSERVATION

ログが失われている状態では、

・同じ崩れが繰り返される
・変化が蓄積されない
・回復が一時的になる
・原因が曖昧になる

これは、状態が観測されず、
循環が断たれている状態である。

逆にログが機能するとき、

・変化の兆候が早期に観測される
・回復の経路が再利用される
・失敗が構造として理解される
・状態が連続的に把握される

ログとは、
状態の「見える化」ではなく、
状態の「通りの保持」である。

# CONNECTION

ログ語彙は以下と接続する。

・MC-014（観測）
・MC-013（循環）
・MC-015（ログ構造）
・CL-000～004（ログ実装）
・BP系（身体観測）

また、

身体ログ → 関係ログ → 文明ログ

というスケール連鎖により、
ログは文明設計の基盤となる。
