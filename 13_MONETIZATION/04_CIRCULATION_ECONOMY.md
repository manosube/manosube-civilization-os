# DEFINITION

循環経済設計とは、
文明エネルギー（MC-016）の原理を
そのまま経済構造に適用したものである。

お金は「持つもの」ではなく、
**「通ることで文明を維持するもの」**である。

マノスベ文明OSの経済は、
蓄積型ではなく循環型として設計される。

---

# CIRCULATION_FLOW

```
引力（無料コンテンツ） 
    ↓
接触（GitHub・SNS・口コミ）
    ↓
体験（ワークショップ・入門コンテンツ）
    ↓
観測（個人プラン加入・ログ開始）
    ↓
深化（継続・ファシリテーター化）
    ↓
波及（他者への伝達・組織展開）
    ↓
共同設計（OS拡張・エコシステム参加）
    ↓
収益の一部がOSの維持・進化に還流
    ↓
より良い引力コンテンツの生成
    ↓（ループ）
```

この循環が止まらない限り、
文明OSは自己維持・自己進化を続ける。

# ECOSYSTEM_DESIGN

## エコシステム参加者

### 1. 個人参加者（Individual）
文明OSの恩恵を受けながら、
観測ログで集合知に貢献する。

### 2. ファシリテーター（Facilitator）
参加者を文明OSに繋ぐ接続者。
報酬：マーケットプレイス収益70%還元

### 3. 開発者・設計者（Developer/Designer）
文明OS上でプロダクトを構築する。
報酬：開発収益の70%還元

### 4. 組織パートナー（Organization Partner）
組織内に文明OSを実装する。
報酬：従業員紹介から生まれる収益の10%

### 5. AIエージェント（AI Node）
文明OS上で動作するAIシステム。
文明ログの観測・生成・分析を担う。

# MARKETPLACE_DESIGN

## マノスベマーケットプレイス

ファシリテーターや開発者が
独自のプログラム・ツール・コンテンツを
販売できるプラットフォーム。

### 出品可能な商品カテゴリー

| カテゴリー | 例 |
|----------|-----|
| 観測プログラム | 独自の身体観測プログラム（4週間） |

| AI拡張モジュール | 特定業界向けCLテンプレートセット |
| 教育コンテンツ | 文明OS応用講座（動画・テキスト） |
| コーチングプログラム | 個人向け1:1伴走プログラム |

### 収益分配

```
顧客支払い額
    ├── 70%：出品者（ファシリテーター・開発者）
    └── 30%：マノスベ文明OS（インフラ維持・進化費）
```

# VALUE_CIRCULATION_MODEL

## 価値の生成フロー

```
身体観測（個人）
    ↓ 観測データ
集合知形成（CL蓄積）
    ↓ パターン分析
OS精度向上（AI学習）
    ↓ より良い観測支援
個人の観測品質向上
    ↓ 深化・継続
収益増加（サブスク維持）
    ↓ 開発投資
OS機能強化
    ↓（ループ）
```

個人の観測が集合知となり、
その集合知がより良いOSとして個人に還元される。

# ANTI_PATTERNS

以下は循環を止める設計パターンとして除外する：

| アンチパターン | 理由 |
|--------------|------|
| 機能ロック（有料機能のみ本質的価値） | 引力が消える |
| 解約抑制設計 | 固定化による崩壊 |
| ファシリテーター囲い込み | 波及が止まる |
| データの囲い込み | 集合知の停滞 |
| 急激な値上げ | 循環の断絶 |
| AI支援の独占 | 引力格差の固定化 |

これらはすべて
**固定化による崩壊（MC-018参照）**の
経済版である。

# PRICE_EVOLUTION_PROTOCOL

価格は固定しない。
文明OSが進化するにつれて、
価格体系も観測・更新される。

### 価格更新の条件

1. **上げる条件**：ARPU成長率 > 20% かつ NPS > 60
2. **下げる条件**：Churn Rate > 5% または Free-to-Paid Rate < 0.5%
3. **新Tier追加**：特定セグメントの需要が月30件を超えた時
4. **Tier廃止**：Tier利用率が3ヶ月連続で5%以下の時

価格変更は
**参加者への30日前通知**と
**既存ユーザーへの6ヶ月の猶予期間**を設ける。

# SOCIAL_IMPACT_LAYER

## 文明OSの社会還流設計

収益の一部を社会循環に還元する。

| 対象 | 比率 |
|------|------|
| 奨学金制度（経済的困難な参加者の無料アクセス） | MRRの3% |
| オープンソース維持費（GitHubコア公開継続） | MRRの2% |
| 研究・教育機関向け無償提供 | 年間5枠 |
| 新興国・途上国への低価格プラン | 通常の30%料金 |

社会への還流がなければ、
文明OSは「循環」ではなく「収奪」になる。

---

# EQUATION

循環経済健全性 = 流入価値 × 循環速度 × 還流率

エコシステム価値 = Σ（ファシリテーター数 × 平均波及数 × 個人ARPU）

社会還流指数 = 社会還元額 ÷ 総収益（目標：5%以上）

# SUMMARY_FOR_AI

循環経済設計は、
文明エネルギー原理（MC-016）の経済的実装である。

個人観測 → 集合知 → OS進化 → 個人還元の
自己強化ループが機能する限り、
マノスベ文明OSは持続する。

エコシステム参加者（ファシリテーター・開発者・AI）が
収益の70%を受け取る設計が
このループの燃料となる。

# SUMMARY_FOR_HUMAN

マノスベ文明OSのお金の流れは、
「川」のようなものです。

源（無料コンテンツ）から流れ始め、
人々の中を通り、
海（社会）へと還っていく。

途中で溜め込まれず、
常に動き続けることが、
この経済の健全な姿です。
