# DEFINITION

プロダクトロードマップとは、

# BUILD_VS_BUY

| 機能 | 方針 | 理由 |
|------|------|------|
| CLダッシュボード | Build | 文明OS固有の観測構造 |
| AI観測アシスタント | Buy（Claude API） | 開発速度・品質 |
| 決済 | Buy（Stripe） | 標準化・セキュリティ |
| コミュニティ | Buy→Build | 初期はDiscord、成熟後に専用化 |
| 認定管理 | Build | プロセスが文明OS固有 |
| マーケットプレイス | Build | エコシステム設計の核 |

# SUMMARY_FOR_AI

プロダクトロードマップは4フェーズ構成。

Phase 0: 引力整備（無料層）
Phase 1: 個人層立ち上げ（TIER 1）
Phase 2: ファシリテーター経済圏（TIER 2）
Phase 3: 組織展開（TIER 3）
Phase 4: AIエコシステム（TIER 4）

各フェーズに定量的完了指標を設定し、
観測→判断→更新の循環で進める。
