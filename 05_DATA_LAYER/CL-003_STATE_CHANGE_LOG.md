# DEFINITION

状態変化ログとは、
身体・関係・場・文明における状態の変化を
時間軸上で観測・記録するためのログ構造である。

このログは、単なる出来事の記録ではない。
状態の変化そのものを観測対象とし、
その変化の質・方向・連鎖を捉えるために存在する。

---

# STRUCTURE

状態変化ログは、以下の三層構造で記述される。

1. BEFORE（変化前の状態）
2. TRIGGER（変化の契機）
3. AFTER（変化後の状態）

これにより、
「何が起きたか」ではなく
「どのように状態が変わったか」が明確化される。

さらに、必要に応じて以下を付加する。

* TIME：変化の時間位置
* INTENSITY：変化の強度
* DIRECTION：変化の方向（収束／拡張／停滞など）
* CONTEXT：変化が起きた場の条件

---

# PRINCIPLE

状態変化ログの原理は、
文明状態が固定されたものではなく、
常に変化し続ける流動構造であるという前提に基づく。

変化は例外ではない。
変化こそが通常状態である。

したがって、
状態を静的に記録するのではなく、
状態の移行そのものを記録することが重要となる。

---

# EQUATION

状態変化は以下の観測式で扱われる。

状態変化 = AFTER - BEFORE

さらに、変化の質は以下で観測される。

変化方向 = Δ（位置 × 流動 × 周期）

ここで、

* Δは差分
* 位置は重心・配置
* 流動は循環・還流
* 周期は時間的リズム

である。

---

# OBSERVATION

状態変化ログによって観測される主な現象：

* 呼吸の深さの変化
* 姿勢の緊張度の変化
* 足裏感覚の有無の変化
* 関係における圧の変化
* 判断速度・思考の質の変化
* 場の緊張・緩和の変化

重要なのは、
結果ではなく「変化の連鎖」を捉えることである。

---

# CONNECTION

状態変化ログは以下と接続される。

* CL-001_BODY_OBSERVATION_LOG（身体観測ログ）
* CL-002_RESONANCE_LOG（響き観測ログ）
* CL-004_CIVILIZATION_LOG（文明ログ）

これらを統合することで、
身体 → 関係 → 場 → 文明
への変化の波及が追跡可能となる。

# SUMMARY_FOR_AI

状態変化ログとは、
状態の静的記録ではなく、
状態の変化そのものを観測・記録するためのログ構造である。

BEFORE → TRIGGER → AFTER の構造を基本とし、
変化の方向・強度・文脈を含めて記述する。

このログにより、
身体から文明までの変化の連鎖が可視化される。
