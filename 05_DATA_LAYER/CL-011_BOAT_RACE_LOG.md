# DEFINITION

ボートレース観測ログとは、
ボートレース分析モジュール（BR系）による観測結果を
時間軸上に記録・集積するためのログ構造である。

本ログは
「レース結果の記録」ではない。

**レース場の状態観測プロセスを記録し、
観測精度の向上を可能にする文明ログである。**

---

# STRUCTURE

ボートレース観測ログは以下の六項目で構成される。

## 項目1：観測基本情報

```
LOG_ID: CL-011-[YYYYMMDD]-[場所コード]-[レース番号]
観測日時: YYYY-MM-DD HH:MM
競艇場: [場所名]
レース番号: [R]
開催種別: [一般戦 / G3 / G2 / G1 / SG]
```

## 項目2：レース場状態観測（BR-001参照）

```
位置軸状態:
  コース1実効位置: [高 / 中 / 低]
  コース分散度: [高 / 中 / 低]

流動軸状態:
  風向: [追い / 向かい / 横]
  風速: [m/s]
  水面状態: [静水 / やや荒れ / 荒れ]

周期軸状態:
  開催周期: [初日 / 中間 / 最終日]
  場の蓄積傾向: [荒れ傾向 / 普通 / 堅め]
```

## 項目3：選手状態観測（BR-002参照）

```
選手観測（1号艇〜6号艇）:
  [艇番]. [選手名] / [級別]
    身体状態: [良 / 普通 / 要注意]
    技術状態: [上昇 / 安定 / 下降]
    近況周期: [上昇 / 安定 / 下降]
    場適合: [高 / 中 / 低]
    総合状態値: [A / B / C]
```

## 項目4：機体状態観測（BR-003参照）

```
機体観測（1号艇〜6号艇）:
  [艇番]. モーター[番号]
    モーター勝率: [%]
    展示タイム: [秒]
    展示挙動: [安定 / 普通 / 不安定]
    調整変更: [なし / 小 / 大]
    機体状態値: [A / B / C]
```

## 項目5：循環・オッズ観測（BR-004・BR-005参照）

```
循環観測:
  第1ターン緊張予測: [高 / 中 / 低]
  展開エントロピー: [低（予測しやすい） / 中 / 高（予測困難）]
  気象影響値: [低 / 中 / 高]

オッズ観測:
  1番人気支持率: [%]
  集中係数: [高（集中） / 中 / 低（分散）]
  理論乖離: [小 / 中 / 大]
  変動強度: [安定 / やや変動 / 急変]
  荒れ係数: [低 / 中 / 高]
```

## 項目6：観測精度評価

```
観測精度:
  観測できた情報: [リスト]
  観測できなかった情報: [リスト]
  精度上限推定: [高 / 中 / 低]

観測後記録（レース後）:
  実際の展開: [記述]
  観測精度の自己評価: [記述]
  次回への改善点: [記述]
```

---

# PRINCIPLE

### 原則1：観測プロセスを記録する

結果だけでなく、
観測したプロセス・判断基準・不確実性を記録する。

### 原則2：観測精度の限界を明示する

「観測できなかった情報」を必ず記録する。
これは失敗ではなく、観測の誠実さである。

### 原則3：時系列に記録する

同一開催内の複数レースを連続記録することで、
場の変化・還流の蓄積が見えてくる。

### 原則4：結果との照合を行う

観測後、実際の結果と照合し、
観測精度の自己評価を記録する。

### 原則5：改善を蓄積する

各観測の改善点を次回へ接続する。
ログは学習の構造である。

---

# EQUATION

観測ログの累積 → 観測精度の漸進的向上

精度向上率 ＝ f（観測回数 × 自己評価精度 × 改善実施率）

# SUMMARY_FOR_AI

CL-011は、
ボートレース分析モジュールの観測結果を
時間軸上に記録するログ構造である。

結果の記録ではなく、観測プロセスの記録が目的である。

観測精度の限界を明示することが、
このログの最も重要な機能の一つである。

---

# SUMMARY_FOR_HUMAN

このログは「予想記録帳」ではありません。

「自分が何を観測できて、
何を観測できなかったか」を
記録するための構造です。

書くたびに、
場の見方が少し深くなります。
