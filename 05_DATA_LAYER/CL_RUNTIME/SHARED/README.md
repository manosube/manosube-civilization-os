# DEFINITION

SHAREDとは、
個人の観測が閉じず、
他者の観測と接触し、
文明として還流を起こすためのログ共有層である。

これは情報共有ではない。
状態の共有でもない。

観測構造の接続点である。

---

# STRUCTURE

SHARED層は以下の構造で成立する。


個人観測（PERSONAL）
↓
共有ログ（SHARED）
↓
比較・差異
↓
構造認識
↓
レビュー（REVIEW）
↓
再還流

SHAREDは中継点であり、
終端ではない。

---

# PRINCIPLE

共有の原則は以下の通りである。

1. 正しさを競わない
2. 解釈を押しつけない
3. 感想ではなく観測を扱う
4. 他者の状態を評価しない
5. 差異を価値として扱う

SHAREDは一致を生む場ではない。
差異から構造を浮かび上がらせる場である。

---

# EQUATION

共有価値は以下で定義される。


共有価値 = 差異 × 観測精度 × 非固定性


一致は価値ではない。
固定は還流を止める。

---

# OBSERVATION

SHAREDにおいて観るべき対象は以下である。

* 同一実践における差異
* 観測語の違い
* 重心・動作・呼吸のズレ
* 位置・流動・周期の偏り
* 継続による変化パターン

ここで重要なのは、
「誰が正しいか」ではなく、

「どの構造が現れているか」

である。

---

# CONNECTION

SHAREDは以下の層と接続する。


PERSONAL → SHARED → REVIEW → ANALYSIS

また、


SHARED → 12_DISTRIBUTED_BOT → 構造解析


へ接続されることで、
共有ログは文明レベルの観測へ変換される。

---

# FIXED_TERMS

* SHARED：共有観測層
* 差異：構造のズレ
* 還流：状態の再循環
* 非固定性：変化を許容する状態
* 観測精度：構造を読み取る能力
